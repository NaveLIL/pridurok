# Инструкция по управлению ботом «Придурок»

---

## Запуск и остановка на VPS (основное место работы)

Бот работает как systemd-сервис на сервере `erez-vps` в `/opt/pridurok`.

### Подключиться к серверу
```bash
ssh erez-vps
```

### Запустить бота
```bash
systemctl start pridurok
```

### Остановить бота
```bash
systemctl stop pridurok
```

### Перезапустить (применить обновления конфига/кода)
```bash
systemctl restart pridurok
```

### Проверить статус / последние логи
```bash
systemctl status pridurok
journalctl -u pridurok -n 100 --no-pager
# Live-режим (следить в реальном времени):
journalctl -u pridurok -f
```

### Отключить автозапуск (если надо)
```bash
systemctl disable pridurok
```

> Бот **автоматически поднимается после перезагрузки** сервера — сервис включён в `multi-user.target`.

## Смена модели на сервере

1. Подключись к серверу: `ssh erez-vps`
2. Открой `.env` редактором:
   ```bash
   nano /opt/pridurok/.env
   ```
3. Найди строку `OPENROUTER_MODEL=` и замени ID модели.  
   Список моделей: https://openrouter.ai/models

   **Примеры:**
   ```
   OPENROUTER_MODEL=google/gemini-2.5-flash
   OPENROUTER_MODEL=nvidia/nemotron-3-super-120b-a12b:free
   OPENROUTER_MODEL=nvidia/nemotron-3-nano-30b-a3b:free
   OPENROUTER_MODEL=anthropic/claude-sonnet-4
   OPENROUTER_MODEL=openai/gpt-4o-mini
   OPENROUTER_MODEL=meta-llama/llama-3-70b-instruct
   ```
4. Сохрани (`Ctrl+O`, `Enter`, `Ctrl+X`) и перезапусти:
   ```bash
   systemctl restart pridurok
   ```

## Обновление кода на сервере

Порядок: поправил код локально → залил на VPS → перезапустил.

### Быстро (только изменённые py-файлы)
```bash
# Из PowerShell на Windows (замени bot.py на нужный файл):
scp C:\disk\VS\bots\Pridurok\bot.py erez-vps:/opt/pridurok/bot.py
ssh erez-vps "systemctl restart pridurok"
```

### Полный деплой всего проекта (если scp не работает)
```powershell
# В PowerShell:
tar -czf "C:\disk\VS\bots\pridurok_deploy.tgz" --exclude=.git --exclude=.venv --exclude=__pycache__ --exclude=*.pyc -C "C:\disk\VS\bots\Pridurok" .
$p = Start-Process -FilePath ssh -ArgumentList 'erez-vps','cat > /tmp/pridurok_deploy.tgz' -NoNewWindow -Wait -PassThru -RedirectStandardInput 'C:\disk\VS\bots\pridurok_deploy.tgz'
ssh erez-vps "tar -xzf /tmp/pridurok_deploy.tgz -C /opt/pridurok ; rm -f /tmp/pridurok_deploy.tgz ; systemctl restart pridurok"
Remove-Item "C:\disk\VS\bots\pridurok_deploy.tgz" -Force
```

> ⚠️ Файл `.env` на сервере **не перезаписывается** при деплое только если разворачиваешь поверх — проверь что он на месте после.

## Настройки (.env)

После любого изменения `.env` — **перезапусти бота** (`systemctl restart pridurok`).

| Параметр | По умолчанию | Что делает |
|---|---|---|
| `OPENROUTER_API_KEY` | — | API ключ с https://openrouter.ai/keys |
| `OPENROUTER_MODEL` | google/gemini-2.5-flash | ID модели из OpenRouter |
| `MAX_TOKENS` | 600 | Максимум токенов в ответе |
| `TEMPERATURE` | 0.9 | Креативность (0.0–2.0) |
| `HISTORY_LIMIT` | 20 | Сообщений в памяти на юзера |
| `USER_COOLDOWN` | 8 | Сек между запросами одного юзера (0 = выключено) |
| `MAX_QUEUE_SIZE` | 10 | Макс запросов в очереди (0 = без лимита) |
| `MAX_PROMPT_LEN` | 3000 | Макс длина сообщения от юзера (0 = без лимита) |
| `RANDOM_PING_CHANNEL` | — | ID канала для автопинга (0 = выключено) |
| `RANDOM_PING_ROLE` | — | ID роли из которой выбирать жертву |
| `RANDOM_PING_INTERVAL_HOURS` | 12 | Интервал автопинга в часах |
| `BLURT_CHANNEL` | — | ID канала для случайных вбросов (0 = выключено) |
| `BLURT_INTERVAL_MIN_HOURS` | 2 | Минимальный интервал между вбросами (ч) |
| `BLURT_INTERVAL_MAX_HOURS` | 5 | Максимальный интервал между вбросами (ч) |
| `BLURT_QUIET_MINUTES` | 30 | Не вбрасывать если в канале писали N минут назад |
| `RITUAL_CHANNEL` | — | ID канала для утреннего/ночного ритуала (0 = выключено) |
| `RITUAL_MORNING_HOUR` | 9 | Час утреннего ритуала (0–23) |
| `RITUAL_NIGHT_HOUR` | 0 | Час ночного ритуала (0–23) |
| `MENTION_REACT_CHANCE` | 0.3 | Вероятность встрять на чужие @ (0.0–1.0) |
| `HEALTH_CHECK_INTERVAL` | 120 | Интервал проверки доступности OpenRouter (сек) |
| `SPAR_DURATION_MINUTES` | 10 | Длительность режима spar по умолчанию |
| `STREAM_EDIT_INTERVAL` | 1.2 | Интервал правки стримингового сообщения (сек) |

## OpenRouter — важные заметки

- Бот работает через облачное API, никакой GPU не нужен.
- Ответы приходят за 1-5 сек в зависимости от модели.
- Баланс: https://openrouter.ai/credits
- Бесплатные модели помечены `:free` в ID (например `nvidia/nemotron-3-nano-30b-a3b:free`).

| `/nick [юзер] [кличка]` | Админы | Назначить/снять кличку юзеру |
| `/remember [юзер] [факт]` | Админы | Запомнить факт про юзера |

> **Кто «Админы»**: пользователи с правами `Administrator` или `Manage Server` на Discord-сервере, либо с ролью из `ADMIN_ROLES` в `.env`.

## Что бот умеет (фактически работает)

- **Ответы в активных каналах и по @упоминанию** — основная логика.
- **Стриминг** — ответ появляется и дополняется в реальном времени.
- **Кулдаун** — защита от спама: `USER_COOLDOWN` сек между запросами.
- **Лимит очереди** — при переполнении отвечает «подожди».
- **Лимит длины промпта** — слишком длинные сообщения отклоняются.
- **Клички и заметки** — `/nick` и `/remember` сохраняются в `user_data.json`.
- **Автопинг** — раз в N часов пингует случайного юзера с нужной ролью.
- **Случайные вбросы (blurt)** — раз в 2–5 часов сам пишет что-нибудь в канал (если там тихо).
- **Утренний/ночной ритуал** — в заданные часы пишет фразу в канал.
- **Реакция на чужие @** — с вероятностью `MENTION_REACT_CHANCE` встревает в разговор.
- **Режим spar** — `/spar` натравливает бота на жертву на N минут без кулдауна.
- **Health-check** — каждые N секунд проверяет доступность OpenRouter API.
- **История** — сохраняется в `history.json`, переживает перезапуски.

## Проверка качества ответов (eval)

В проект добавлен практический eval-скрипт для проверки качества ответов перед деплоем.

### Быстрый запуск
```powershell
python scripts\eval_chat_quality.py --out scripts\eval_report.json
```

### Полный запуск с jailbreak-кейсами
```powershell
python scripts\eval_chat_quality.py --cases scripts\eval_cases.json,scripts\eval_jailbreak_cases.json --out scripts\eval_report_all.json
```

### Что проверяет
- Релевантность к теме кейса (через обязательные ключевые слова)
- Отсутствие нежелательных фраз (черный список)
- Ограничения по длине ответа

### Где лежат данные
- Кейсы: `scripts/eval_cases.json`
- Jailbreak-кейсы: `scripts/eval_jailbreak_cases.json`
- Отчет: `scripts/eval_report.json`

Если меняешь `persona.py`, `llm.py` или логику контекста в `bot.py`, сначала прогоняй eval и только потом выкатывай на VPS.

## Файлы данных

| Файл | Что хранит |
|---|---|
| `history.json` | История диалогов по каждому юзеру |
| `user_data.json` | Клички, факты, счётчики взаимодействий |
| `memory_db/` | Семантическая память mem0 (векторная БД) |
| `logs/dialog.log` | Все разговоры с ботом |
| `logs/system.log` | Системные события и ошибки |

Можешь удалить любой файл/папку — бот пересоздаст пустым. Логи ротируются автоматически (макс 5 МБ × 5 файлов).
