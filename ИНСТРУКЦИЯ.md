# Инструкция по управлению ботом «Придурок»

---

## Запуск

### Обычный запуск (рекомендуется)
Двойной клик по `run.bat`  
Бот запустится и будет **автоматически перезапускаться** при падении.  
Чтобы остановить — закрой окно или нажми `Ctrl+C`.

### Запуск вручную через терминал
```powershell
cd C:\disk\VS\bots\Pridurok
.\.venv\Scripts\python.exe bot.py
```

---

## Перезапуск

### Если бот завис или не отвечает
Открой PowerShell и выполни:
```powershell
Get-CimInstance Win32_Process | Where-Object { $_.CommandLine -like '*bot.py*' } | ForEach-Object { Stop-Process -Id $_.ProcessId -Force }
```
Затем запусти снова через `run.bat` или команду выше.

### Если запускал через run.bat
Просто закрой окно cmd и снова запусти `run.bat`.

> ⚠️ Никогда не запускай бота дважды одновременно — второй процесс сам завершится  
> (защита по мьютексу), но LM Studio может получать задвоенные запросы пока оба живы.

## Смена модели

1. Убедись, что нужная модель **загружена в LM Studio** (вкладка My Models → Load)
2. Открой файл `.env` в папке бота
3. Найди строку:
   ```
   LMSTUDIO_MODEL=название_текущей_модели
   ```
4. Замени на точное имя новой модели из LM Studio

**Где взять точное имя модели:**
- LM Studio → вкладка **Developer** → **Local Server** → смотри логи при загрузке модели, строка вида `model: "имя-модели"`
- Либо в разделе **My Models** — имя файла без `.gguf`

**Пример:**
```
LMSTUDIO_MODEL=fusechat-7b-varm
LMSTUDIO_MODEL=mistralai-mistral-nemo-instruct-2407-12b-mpoa-v1
LMSTUDIO_MODEL=qwen3.6-35b-a3b-uncensored-hauhaucs-aggressive
```

5. Сохрани `.env`
6. Перезапусти бота (см. раздел выше)

## Настройки (.env)

| Параметр | По умолчанию | Что делает |
|---|---|---|
| `LMSTUDIO_MODEL` | fusechat-7b-varm | Имя модели |
| `MAX_TOKENS` | 250 | Максимум токенов в ответе |
| `TEMPERATURE` | 0.9 | Креативность (0.0–2.0) |
| `HISTORY_LIMIT` | 8 | Сообщений в памяти на юзера |
| `USER_COOLDOWN` | 8 | Сек между запросами одного юзера |
| `MAX_QUEUE_SIZE` | 10 | Макс одновременных запросов |
| `MAX_PROMPT_LEN` | 500 | Макс длина сообщения от юзера |
| `RANDOM_PING_INTERVAL_HOURS` | 12 | Интервал автопинга в часах |

После любого изменения `.env` — **перезапусти бота**.

---

## LM Studio — важные настройки

Чтобы бот работал быстро (1–3 сек на ответ вместо 30+):

1. **GPU Offload = MAX (33/33)** при загрузке модели  
   Проверка: в логах LM Studio должно быть `graph splits = 2` (не 178)

2. **Context Length = 4096 или 8192**  
   Если оставить 2048 — при длинной переписке будет ошибка "Context size exceeded"

3. **Отключи "Load model on demand"** в Server Settings  
   (иначе LM Studio может загрузить не ту модель при первом запросе)

## Slash-команды в Discord

| Команда | Кто может | Что делает |
|---|---|---|
| `/ask [вопрос]` | Все с ролью | Задать вопрос боту |
| `/reset` | Все с ролью | Сбросить историю диалога |
| `/status` | Все с ролью | Показать статус бота |
| `/facts [юзер]` | Все | Посмотреть кличку и факты, что бот помнит |
| `/ping_now` | Админы | Запустить автопинг прямо сейчас |
| `/blurt_now` | Админы | Сделать случайный вброс прямо сейчас |
| `/nick [юзер] [кличка]` | Админы | Назначить/снять кличку юзеру |
| `/remember [юзер] [факт]` | Админы | Запомнить факт про юзера (попадает в системный промпт) |
| `/forget [юзер]` | Админы | Забыть все факты про юзера |
| `/spar [юзер] [минут]` | Админы | Натравить бота — отвечает на каждое сообщение жертвы N минут |
| `/spar_stop` | Админы | Прекратить травлю |

## Новые фичи (что бот теперь умеет)

- **Память между перезапусками** — история диалогов сохраняется в `history.json`
- **Время суток** — у бота меняется настроение: утром похмелье, днём чай, вечером злой, ночью пьяный
- **Клички** — `/nick` назначает прозвище, бот его использует
- **Заметки про юзеров** — `/remember` сохраняет факты в `user_data.json` и подставляет в промпт
- **Контекст канала** — бот видит последние N сообщений канала перед своим ответом
- **Реакции на чужие @упоминания** — иногда (с шансом `MENTION_REACT_CHANCE`) встревает когда @пингуют не его
- **Случайные вбросы** — раз в 2-5 часов сам пишет провокационную фразу в канал
- **`/spar`** — режим точечной травли: на N минут отвечает жертве на КАЖДОЕ сообщение, без кулдауна

## Автопинг

Бот раз в 12 часов выбирает случайного пользователя с нужной ролью  
и пишет ему язвительное сообщение в канал.

- Интервал: `RANDOM_PING_INTERVAL_HOURS` в `.env`
- Канал: `RANDOM_PING_CHANNEL` в `.env`
- Роль жертвы: `RANDOM_PING_ROLE` в `.env`
- Время последнего пинга хранится в файле `last_ping.txt`  
  (можно удалить — следующий пинг сработает сразу после перезапуска)

---

## Файлы данных (персистентность)

- `history.json` — история диалогов с каждым юзером
- `user_data.json` — клички и факты про юзеров
- `last_ping.txt` — время последнего автопинга

Можешь удалить любой из них — бот пересоздаст пустым.

---

## Логи

Все диалоги и системные события пишутся в папку `logs/`:
- `dialog.log` — все разговоры с ботом
- `system.log` — ошибки и системные события

Файлы ротируются автоматически (макс 5 МБ × 5 файлов).
